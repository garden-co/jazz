<Slide>
# Metrics to judge a stack by

- Lines of code
- Layers of infra BS
- Light-milliseconds

<Notes>
And the three metrics I want you to keep in mind when
you judge a stack are

- lines of code, especially those that don't have anything to
  do with what makes your app unique
- layers of infra bullshit, so how many things do you have to
  deploy and integrate with each other
- light-milliseconds that user edits have to travel before they
  become visible
    </Notes>
</Slide>

# Traditional backend

Lines of code: +++++++++++++++

Layers:

- backend/external
  - Database, ORM, cache, connection pooler
  - backend framework, auth logic
  - queues, pubsub, workers, durable streams
  - auth provider, blob storage
- frontend
  - data fetching, websocket client
  - query cache w/ optimistic updates, ui state
  - client/rendering framework

Light milliseconds: 100ms -> backend -> 30ms db (auth) -> 30ms db (data) -> 100ms client

<Notes>
This is our starting situation, and this is exactly what makes
the backend part so unfun:
