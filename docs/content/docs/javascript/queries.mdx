Jazz provides a type-safe query builder for filtering and subscribing to data. The API is generated from your SQL schema.

## Query Builder Pattern

Jazz uses a fluent query builder pattern rather than raw SQL strings:

```typescript title="queries/basic.ts"
import { app } from '../generated/client';
import { useAll } from '@jazz/react';

// Subscribe to all tasks
const [tasks, loading] = useAll(app.tasks);

// With filtering
const [activeTasks, loading] = useAll(
  app.tasks.where({ completed: false })
);

// With related data
const [tasksWithUsers, loading] = useAll(
  app.tasks
    .where({ completed: false })
    .with({ user: true })
);
```

## Filtering with where()

The `where()` method accepts an object with field conditions:

```typescript
// Simple equality
app.tasks.where({ status: 'open' })

// Multiple conditions (AND)
app.tasks.where({
  status: 'open',
  priority: 'high'
})

// Filter by reference
app.tasks.where({ project: projectId })

// Filter by related data existence
app.issues.where({
  IssueAssignees: { some: { user: userId } }
})
```

## Including Related Data with with()

Use `with()` to include referenced data in results:

```typescript
// Include single reference
app.tasks.with({ project: true })

// Include nested references
app.issues.with({
  project: true,
  IssueLabels: { label: true },
  IssueAssignees: { user: true }
})
```

## Direct Subscriptions

For vanilla JavaScript (outside React), use `subscribeAll`:

```typescript title="queries/subscriptions.ts"
import { app } from '../generated/client';

// Subscribe to query results
const unsubscribe = app.tasks
  .where({ completed: false })
  .subscribeAll(db, (tasks) => {
    console.log('Active tasks:', tasks);
  });

// Clean up when done
unsubscribe();
```

## Supported Query Features

The current implementation supports:

- Filtering with equality conditions (AND logic)
- Loading related data through foreign key references
- Real-time subscriptions with automatic updates
- LIMIT and OFFSET for pagination
- Full TypeScript type inference from schema

### Not Yet Implemented

The following SQL features are planned but not yet available:

- ORDER BY (results currently ordered by ObjectId/insertion time)
- GROUP BY and HAVING
- Aggregate functions (COUNT, SUM, AVG, MIN, MAX)
- LEFT/RIGHT/OUTER JOINs (only INNER JOIN)
- Complex subqueries
- Raw SQL queries from JavaScript
