## Core Concepts

### Branch Naming

Branches encode environment and schema version:

```
[env]-[schemaVersion]-[userBranch]
```

- **schemaVersion**: Full DescriptorId (26-char Crockford Base32 ObjectId)
- **Example**: `prod-01JGXYZ123ABC456DEF789GHIJ@v1-main`

Rows have branches per environment and schema version. Migrations fork branches (v2 branches fork from v1, preserving history).

### Descriptors

Descriptors use regular ObjectIds (not content-addressed hashes):
- Server is authoritative for schemas
- Schema deduplication unlikely to matter in practice
- Parent pointer DAG provides versioning semantics
- Makes schemas "just objects" like everything else

### Bidirectional Lenses

Lenses are auto-generated from schema diffs as SQL transforms:

```rust
enum Lens {
    Rename { from: String, to: String },
    Add { column: String, default: SqlExpr },
    Remove { column: String },
    Transform { column: String, expr: SqlExpr },
}
```

- Auto-generated from schema diff
- "Ignore row" default when transform impossible
- SQL expressions for custom cases

### Migration Execution

1. Developer defines migration, deploys via CLI (`jazz migrate push`)
2. Server executes eagerly (transforms all data on deploy)
3. Clients execute locally on unsynced data
4. Server rejects unknown branches with exponential backoff until schema deployed

## Sync Behavior

- Server applies lenses for query evaluation (filtering, index lookup)
- Server syncs raw rows as-is (preserves source of truth)
- Clients apply lenses locally

## Multi-Schema Rows

Rows can exist on multiple schema branches independently:
- Writer clients create explicit merge commits (cross-branch parents)
- Read-only clients preview merge
- Per-column merge strategy (defined in schema, LWW default)

## Server Architecture

| Mode | Description |
|------|-------------|
| Standalone | Single app, one catalogue |
| Jazz Cloud | Multi-tenant with namespace separation + per-tenant encryption |

1:1 relationship between app and catalogue. Environments (dev/staging/prod) are internal to catalogue via branch namespacing.

## CLI

```bash
jazz migrate status <table>              # Show current schema from server
jazz migrate diff <table> -f schema.sql  # Preview schema changes
jazz migrate push <table> -f schema.sql  # Deploy migration
```
