Jazz uses a commit-based sync protocol inspired by git. Each object has its own commit graph that syncs independently.

## Implementation Status

**Implemented:**
- Protocol types: `SubscribeRequest`, `PushRequest`, `SseEvent`, etc.
- Binary serialization for all message types
- Negotiation logic: frontier comparison, commit discovery
- `SyncClient<E: ClientEnv>` - generic client with transport abstraction
- `SyncServer<E: ServerEnv>` - server-side sync logic
- HTTP handlers: `/sync/subscribe`, `/sync/push`, `/sync/unsubscribe`, `/sync/reconcile`
- SSE streaming with keep-alive
- IndexedDB persistence for browser (groove-wasm)
- `SyncStateStore` for persistent unsynced object tracking

**Not yet implemented:**
- Query result tracking with multi-query reference counting
- Reactive query updates when row changes affect results
- Reconnection handling with exponential backoff
- Push all unsynced objects on reconnect

## Overview

Each object in Jazz has a commit graph. Sync involves:

1. **Exchanging commit hashes**: What commits do we each have?
2. **Sending missing commits**: Ship commits the other side lacks
3. **Merging**: Combine concurrent changes using per-column LWW

## Transport

Sync uses HTTP POST + Server-Sent Events (SSE):

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/sync/subscribe` | POST | Subscribe to objects, returns SSE stream |
| `/sync/push` | POST | Push local commits to server |
| `/sync/unsubscribe` | POST | Unsubscribe from objects |
| `/sync/reconcile` | POST | Reconcile frontiers after reconnect |

## Conflict Resolution

When commits have different parents (concurrent edits):

1. **Detect**: Branch has multiple frontier commits (tips)
2. **Auto-resolve**: Per-column last-writer-wins merge
3. **Result**: Merged commit with values from highest-timestamp per column

## Merge Strategy

| Scenario | Resolution |
|----------|------------|
| Same column edited | Higher timestamp wins |
| Different columns edited | Both changes kept |
| Timestamp tie | Lexicographically higher CommitId wins |
