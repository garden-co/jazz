Plain TypeScript uses the low-level client directly. There is no provider or hooks: you create a
`Db` with `createDb`, then read, subscribe, and write through it. See
[Overview](/docs/reference/supported-frameworks/overview) for the cross-framework API table.

## Client creation

`createDb` returns a `Db` handle. Pass an `appId`, and a `secret` for
[local-first auth](/docs/auth/local-first-auth) (omit it for anonymous, read-only mode).

## Sessions and identity

Read the current session and user ID with `db.getAuthState()`, and react to changes with
`db.onAuthChanged`. See [Sessions](/docs/auth/sessions).

## Local-first auth

Plain TypeScript has no auth hook: read or create the secret directly with `browserAuthSecretStore`
and pass it to `createDb`. See [Local-first auth](/docs/auth/local-first-auth) for the full flows,
including recovery passphrase and passkey backup.
