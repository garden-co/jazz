Jazz's React Native bindings live in `jazz-tools/react-native`. `JazzProvider`, `useDb`,
`useSession`, and `useAll` work exactly as they do in
[React](/docs/reference/supported-frameworks/react/reference)&hairsp;—&hairsp;that page is the
reference for the component API. This page covers what is specific to native.

## Packages

- `jazz-tools/react-native`&hairsp;—&hairsp;the provider and hooks (`JazzProvider`, `useDb`,
  `useSession`, `useAll`). This is all a bare React Native app needs.
- `jazz-tools/expo`&hairsp;—&hairsp;Expo's local-first auth: `ExpoAuthSecretStore` (backed by
  `expo-secure-store`) and `useLocalFirstAuth`.
- `jazz-rn`&hairsp;—&hairsp;the native runtime. It is mandatory alongside `jazz-tools` in any
  React Native app.

<Callout type="info">
  Bare React Native (no Expo) is supported directly: import the provider and hooks from
  `jazz-tools/react-native` and supply your own secret store. The `jazz-tools/expo` package is only
  the Expo-flavoured local-first auth on top.
</Callout>

## Native runtime

React Native uses a separate native runtime adapter&hairsp;—&hairsp;no web worker and no OPFS.
Local persistence is provided by the native embedded backend rather than browser storage. See
[Internals](/docs/reference/internals#react-native).

## Local-first auth

`useLocalFirstAuth()` from `jazz-tools/expo` loads the device secret from secure storage (or
generates one on first run). See [Local-first auth](/docs/auth/local-first-auth).
