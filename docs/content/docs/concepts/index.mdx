---
title: What is Jazz?
description: A local-first database that syncs
---

Jazz is a database that runs in your app and syncs automatically. Data lives locally first, so your UI is always fast and works offline. Changes sync to a server and other clients in the background.

## The Core Idea

Instead of fetching data from a server, you read and write to a local database. Jazz handles syncing behind the scenes:

```typescript
// Subscribe to tasks - data is local, updates are instant
const [tasks] = useAll(app.tasks.where({ completed: false }));

// Create a task - no API call, instant UI update
app.tasks.create(db, { title: "Buy groceries", completed: false });
```

The sync server is a peer, not the source of truth. If you're offline, everything still works.

## Key Features

**Instant UI** - Data is local, so reads and writes are fast. No loading spinners for cached data.

**Offline by default** - Full functionality without network. Changes sync when you reconnect.

**SQL interface** - Define tables with SQL, query with familiar syntax.

**Incremental queries** - Subscribe to query results that update efficiently when data changes.

**Automatic sync** - Changes propagate to server and other clients automatically.

**Conflict resolution** - Concurrent edits merge using per-column last-writer-wins.

## How It Works

Under the hood, each row in your database is an "Object" with a git-like commit history. This enables:

- **Fine-grained sync** - Only changed rows sync, not entire tables
- **Per-row versioning** - Each row tracks its own history
- **Automatic merging** - Concurrent edits to different columns merge cleanly

When two clients edit the same row:

1. Both edits apply locally and sync to the server
2. The server broadcasts both changes
3. Each client merges using per-column last-writer-wins
4. The highest timestamp per column wins
