Jazz queries are live subscriptions that update automatically when data changes. You subscribe once, and your UI stays in sync.

## Basic Queries

Use `useAll` to subscribe to multiple rows:

```typescript
import { useAll } from '@jazz/react';
import { app } from './generated/client';

function TaskList() {
  const [tasks, loading] = useAll(app.tasks);

  if (loading) return <div>Loading...</div>;

  return (
    <ul>
      {tasks.map(task => (
        <li key={task.id}>{task.title}</li>
      ))}
    </ul>
  );
}
```

Use `useOne` to subscribe to a single row by ID:

```typescript
import { useOne } from '@jazz/react';

function TaskDetail({ taskId }: { taskId: string }) {
  const [task, loading] = useOne(app.tasks, taskId);

  if (loading) return <div>Loading...</div>;
  if (!task) return <div>Not found</div>;

  return <h1>{task.title}</h1>;
}
```

## Filtering

Use `where` to filter results:

```typescript
// All incomplete tasks
const [tasks] = useAll(app.tasks.where({ completed: false }));

// Tasks assigned to a specific user
const [myTasks] = useAll(app.tasks.where({ assignee: userId }));
```

Currently supported: equality filters (`=`, `!=`). Range operators coming soon.

## Including Related Data

Use `with` to eagerly load related rows:

```typescript
// Load tasks with their assignee
const [tasks] = useAll(
  app.tasks.with({ assignee: true })
);

// Now task.assignee is a full User object, not just an ID
tasks.map(task => (
  <li key={task.id}>
    {task.title} - {task.assignee?.name}
  </li>
));
```

You can chain `where` and `with`:

```typescript
const [tasks] = useAll(
  app.tasks
    .where({ completed: false })
    .with({ assignee: true })
);
```

## How It Works

When you call `useAll` or `useOne`:

1. Jazz executes the query against local data
2. Returns results immediately (or loading state if not yet loaded)
3. Subscribes to changes affecting the query
4. Re-renders your component when results change

This is different from traditional "fetch on mount" patterns. Your query stays live for as long as the component is mounted.

## Incremental Updates

Jazz doesn't re-run the entire query when data changes. Instead, it tracks which rows contribute to your results and updates incrementally:

- Row added that matches filter → add to results
- Row updated, no longer matches → remove from results
- Row updated, still matches → update in place

This makes large result sets efficient - only the delta is processed.

## Not Yet Implemented

The following SQL features are not yet available:

- ORDER BY (results ordered by creation time)
- Aggregate functions (COUNT, SUM, etc.)
- GROUP BY
- LEFT/RIGHT/OUTER JOINs

See [Queries spec](/docs/internals/queries) for implementation details.
