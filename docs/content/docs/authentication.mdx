title: Authentication
description: Anonymous, demo, and external JWT auth in Jazz 2, from zero-config onboarding to external provider upgrades.

Jazz 2 supports three auth methods:

1. `anonymous`: one synthetic user token per device
2. `demo`: multiple synthetic users per device for local multi-user simulation
3. `external`: bearer JWT validated via JWKS

## 1. Start with Anonymous Auth (Zero Config)

Browser clients default to anonymous local auth when no JWT/backend auth is configured.
On first load, Jazz auto-generates a device token and persists it in `localStorage`.

<Tabs items={["React", "Svelte", "TypeScript"]}>
  <Tab value="React">
    ```tsx
    import { createJazzClient, JazzProvider } from "jazz-tools/react";

    const client = createJazzClient({
      appId: process.env.NEXT_PUBLIC_JAZZ_APP_ID!,
      env: "dev",
      userBranch: "main",
    });

    export function App() {
      return (
        <JazzProvider
          client={client}><TodoApp /></JazzProvider>;
    }
    ```

  </Tab>
  <Tab value="Svelte">

        userBranch: 'main',

If you want to force a specific device token, pass `localAuthToken` as an explicit override:

```ts
const db = await createDb({
  appId: process.env.JAZZ_APP_ID!,
  env: "dev",
  userBranch: "main",
  localAuthToken: "device-token-123", // localAuthMode defaults to "anonymous"
});
```

## 3. Integrate External Auth and Upgrade Existing Users

Once your auth provider is in place, send provider JWTs and link the current local identity to the external identity.

### Framework Upgrade Hook

Use `useLinkExternalIdentity` right after sign-in to preserve anon/demo-created data:

<Tabs items={["React", "Svelte"]}>
  <Tab value="React">
    ```tsx
    import { useState } from "react";
    import { createJazzClient, JazzProvider, useLinkExternalIdentity } from "jazz-tools/react";

    const appId = process.env.NEXT_PUBLIC_JAZZ_APP_ID!;
    const serverUrl = process.env.NEXT_PUBLIC_JAZZ_SERVER_URL!;

    function AuthGate() {
      const [jwtToken, setJwtToken] = useState<string | undefined>();
      const linkExternalIdentity = useLinkExternalIdentity({
        appId,
        serverUrl,
        defaultMode: "anonymous",
      });

      async function onSignedIn(providerJwt: string) {
        await linkExternalIdentity({ jwtToken: providerJwt });
        setJwtToken(providerJwt);
      }

      // const client = await createJazzClient({
      //   appId,
      //   env: "dev",
      //   userBranch: "main",
      //   jwtToken,
      // });

      return (
        <JazzProvider
          client={client}
        >
          <button onClick={() => onSignedIn("<provider-jwt>")}>Sign in</button>
          <TodoApp />
        </JazzProvider>
      );
    }
    ```

  </Tab>
  <Tab value="Svelte">
    ```svelte
    <script lang="ts">
      import { JazzSvelteProvider, useLinkExternalIdentity } from 'jazz-tools/svelte';

      const appId = import.meta.env.JAZZ_APP_ID ?? 'my-app';
      const serverUrl = import.meta.env.JAZZ_SERVER_URL!;

      const linkExternalIdentity = useLinkExternalIdentity({
        appId,
        serverUrl,
        defaultMode: 'anonymous',
      });

      let jwtToken = $state<string | undefined>();

      async function onSignedIn(providerJwt: string) {
        await linkExternalIdentity({ jwtToken: providerJwt });
        jwtToken = providerJwt;
      }

      const config = $derived({
        appId,
        env: 'dev',
        userBranch: 'main',
        jwtToken,
      });
    </script>

    <JazzSvelteProvider {config}>
      {#snippet children({ db })}
        <button onclick={() => onSignedIn('<provider-jwt>')}>Sign in</button>
        <TodoApp />
      {/snippet}
    </JazzSvelteProvider>
    ```

  </Tab>
</Tabs>

### Non-Framework Upgrade Path

Use `JazzClient.linkExternalIdentity(...)` directly:

```ts
const result = await client.linkExternalIdentity({
  jwtToken: providerJwt,
  localAuthMode: "anonymous", // or "demo"
  localAuthToken: "device-token-123",
});

console.log(result.created, result.principal_id);
```

After linking, external JWT sessions resolve to the same principal.

## Session Resolution Order

For both `jazz-tools server` and `jazz-cloud-server`:

1. Backend impersonation (`X-Jazz-Backend-Secret` + `X-Jazz-Session`)
2. External JWT (`Authorization: Bearer <jwt>`)
3. Local auth (`X-Jazz-Local-Mode` + `X-Jazz-Local-Token`)
4. No session

## Server Setup

### `jazz-tools server` (single app)

- Supports external JWT only when `--jwks-url` is configured.
- Local auth mode defaults depend on `NODE_ENV`:
  - `NODE_ENV=production`: anonymous/demo are disabled unless explicitly enabled
    with `--allow-anonymous` / `--allow-demo`
  - non-production (`development`, unset, etc.): anonymous and demo are enabled by
    default
- Supports external identity mapping persistence via `external_identities`.

Example:

```bash
NODE_ENV=production \
jazz-tools server "$JAZZ_APP_ID" \
  --port 1625 \
  --data-dir ./data \
  --allow-demo \
  --jwks-url https://auth.example.com/.well-known/jwks.json \
  --backend-secret "$JAZZ_BACKEND_SECRET" \
  --admin-secret "$JAZZ_ADMIN_SECRET"
```

### `jazz-cloud-server` (multi-tenant)

Per app config supports:

- `jwks_endpoint` (optional; if unset, external JWT auth is disabled)
- `allow_anonymous` (default `true`)
- `allow_demo` (default `true`)

And a persistent `external_identities` mapping table for upgrade/link fallback.
