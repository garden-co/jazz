title: Authentication

Jazz 2 supports three auth methods:

1. `anonymous`: one synthetic user token per device (default local-first onboarding)
2. `demo`: multiple synthetic users per device to simulate multi-user flows
3. `external`: bearer JWT validated via JWKS

## Session Resolution Order

For both `jazz-tools server` and `jazz-cloud-server`:

1. Backend impersonation (`X-Jazz-Backend-Secret` + `X-Jazz-Session`)
2. External JWT (`Authorization: Bearer <jwt>`)
3. Local auth (`X-Jazz-Local-Mode` + `X-Jazz-Local-Token`)
4. No session

## Client Setup

### Anonymous / Demo Local Auth

```ts
import { JazzClient } from "jazz-tools";

const client = await JazzClient.connect({
  appId: process.env.JAZZ_APP_ID!,
  schema,
  serverUrl: process.env.JAZZ_SERVER_URL!,
  localAuthMode: "anonymous", // or "demo"
  localAuthToken: "device-token-123",
});
```

### External JWT Auth

```ts
import { JazzClient } from "jazz-tools";

const client = await JazzClient.connect({
  appId: process.env.JAZZ_APP_ID!,
  schema,
  serverUrl: process.env.JAZZ_SERVER_URL!,
  jwtToken: userJwt,
});
```

### Link Local -> External (Upgrade)

Use the built-in client call path:

```ts
const result = await client.linkExternalIdentity({
  jwtToken: userJwt,
  localAuthMode: "anonymous",
  localAuthToken: "device-token-123",
});

console.log(result.created, result.principal_id);
```

After linking, external JWT sessions resolve to the same principal.

## Demo User Switcher UI

Switcher helpers are backed by `localStorage` and support profile switching in `demo` mode.

<Tabs items={["React", "TypeScript"]}>
  <Tab value="React">
    ```tsx
    import { JazzProvider, SyntheticUserSwitcher, getActiveSyntheticAuth } from "jazz-tools/react";

    const appId = process.env.NEXT_PUBLIC_JAZZ_APP_ID!;
    const active = getActiveSyntheticAuth(appId, { defaultMode: "demo" });

    export function App() {
      return (
        <>
          <SyntheticUserSwitcher appId={appId} defaultMode="demo" />
          <JazzProvider
            config={{
              appId,
              env: "dev",
              userBranch: "main",
              localAuthMode: active.localAuthMode,
              localAuthToken: active.localAuthToken,
            }}
          >
            <TodoApp />
          </JazzProvider>
        </>
      );
    }
    ```
  </Tab>
  <Tab value="TypeScript">
    ```ts
    import { createDb, createSyntheticUserSwitcher, getActiveSyntheticAuth } from "jazz-tools";

    const appId = process.env.JAZZ_APP_ID!;
    const active = getActiveSyntheticAuth(appId, { defaultMode: "demo" });

    createSyntheticUserSwitcher({
      appId,
      container: document.getElementById("auth-controls")!,
      defaultMode: "demo",
    });

    const db = await createDb({
      appId,
      env: "dev",
      userBranch: "main",
      localAuthMode: active.localAuthMode,
      localAuthToken: active.localAuthToken,
    });
    ```
  </Tab>
</Tabs>

Switching profiles persists locally; by default the switcher refreshes the page after a profile change.

## Server Setup

### `jazz-tools server` (single app)

- Supports external JWT only when `--jwks-url` is configured.
- Local auth mode defaults depend on `NODE_ENV`:
  - `NODE_ENV=production`: anonymous/demo are disabled unless explicitly enabled
    with `--allow-anonymous` / `--allow-demo`
  - non-production (`development`, unset, etc.): anonymous and demo are enabled by
    default
- Supports external identity mapping persistence via `external_identities`.

Example:

```bash
NODE_ENV=production \
jazz-tools server "$JAZZ_APP_ID" \
  --port 1625 \
  --data-dir ./data \
  --allow-demo \
  --jwks-url https://auth.example.com/.well-known/jwks.json \
  --backend-secret "$JAZZ_BACKEND_SECRET" \
  --admin-secret "$JAZZ_ADMIN_SECRET"
```

### `jazz-cloud-server` (multi-tenant)

Per app config supports:

- `jwks_endpoint` (optional; if unset, external JWT auth is disabled)
- `allow_anonymous` (default `true`)
- `allow_demo` (default `true`)

And a persistent `external_identities` mapping table for upgrade/link fallback.
