## Datatypes and References

- TypeScript DSL defaults to `col.string`, `col.boolean`, and `col.ref` for relationships.
- SQL keeps the same model via `TEXT`, `BOOLEAN`, and `UUID REFERENCES ...`.
- Use SQL as the alternate path when your project is Rust-first.

For browser files and large uploaded blobs, define the conventional `files` / `file_parts`
tables and store a normal reference to `files` from your app rows. See
[Files & Blobs](/docs/files-and-blobs).

## Schema Defaults

You can declare default values directly in the schema definition with `.default(...)`.

```ts
table("todos", {
  title: col.string(),
  done: col.boolean().default(false),
  tags: col.array(col.string()).default([]),
  ownerId: col.ref("users").optional().default(null),
});
```

Schema defaults are used in two places:

- For inserts, when a caller omits a field.
- In auto-generated lens defaults, when a column is added or removed across schema versions.

### Insert behavior

When you use insert new values:

- Omitted fields use the schema default if present.
- Filds explicitly set to `undefined` are treated the same as omitted.
- `null` is only allowed for nullable columns, and it is not replaced by the default value.

That means schema defaults affect future inserts, not reads of already-stored rows.

### Schema evolution behavior

Schema defaults also feed Jazz's auto-generated lenses:

- When a column is added across schema versions, Jazz uses the column's default for new rows that predate the new column.
- When a column is removed across schema versions, Jazz uses the column's default when auto-generating the reverse/default lens behavior for older schemas.

### Backwards default examples

Backwards defaults define what older-schema clients see for fields dropped in newer schemas.
When new clients write rows without that field, the backward lens supplies the configured default to old clients.
In TypeScript, the same migration stub carries both forward and backward translation behavior.
In SQL, the backward lens file explicitly reintroduces dropped columns with a default.

If the lens is auto-generated, Jazz will first look for schema defaults on the column definition before falling back to heuristic defaults.
