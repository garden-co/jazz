## Current Schema

Every Jazz app uses a root `schema.ts` as the source of truth, including Rust backends.
`jazz-tools build` validates that file and compiles it into Jazz's internal schema representation.
Jazz no longer generates `schema/app.ts` or `current.sql`.

## Datatypes and References

- Use `col.string()`, `col.boolean()`, `col.int()`, `col.timestamp()`, and `col.bytes()` for scalar fields.
- Use `col.ref("tableName")` for foreign keys. Jazz infers typed `.include(...)` relations directly from `schema.ts`.
- For browser files and large uploaded blobs, define the conventional `files` / `file_parts` tables and store a normal reference to `files` from your app rows. See [Files & Blobs](/docs/files-and-blobs).
