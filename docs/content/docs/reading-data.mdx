## Filters

### API Reference

- **Behavior:** `where(...)` adds AND-combined predicates to a query.
- **Parameters:** Pass a partial row-shaped object.
- **TypeScript operators:** scalar equality (`done: false`) plus operator objects like `{ eq, ne, contains, in, gt, gte, lt, lte, isNull }` where supported by the column type.
- **Rust operators:** use explicit builder methods such as `filter_eq(...)`.

## Sorting

### API Reference

- **Behavior:** `orderBy(...)` applies deterministic ordering to query results.
- **Parameters:** `orderBy(column, direction?)` where `direction` is `"asc"` or `"desc"` in TypeScript.
- **Recommendation:** always sort before paginating.

## Pagination

### API Reference

- **Behavior:** `limit(...)` caps rows; `offset(...)` skips rows.
- **Parameters:** choose a stable page size and compute offset from page index.
- **Recommendation:** combine with a deterministic `orderBy(...)` so page boundaries are stable.

## Resolving References via Includes

### API Reference

- **Behavior:** `include(...)` loads related rows and returns nested objects in one query result.
- **Parameters (TypeScript):** `include({ relationName: true | nestedInclude })`.
- **Nested includes:** pass nested objects to load multi-hop relations.
- **Junction tables:** codegen does not currently synthesize implicit many-to-many shortcuts; model and query the junction table explicitly.

```ts
// Example schema shape:
// project_members(project -> projects.id, user -> users.id)
const memberships = await db.all(
  app.projectMembers.where({ project: projectId }).include({ user: true, project: true }),
);
```
