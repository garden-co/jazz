## Query Graph Nodes

The incremental query system builds a computation graph with these node types:

| Node Type | Purpose |
|-----------|---------|
| `TableScan` | Reads all row IDs from a table |
| `IndexLookup` | Uses reverse index for efficient lookups |
| `Filter` | Applies WHERE conditions |
| `Join` | Combines tables (INNER JOIN) |
| `ArrayAggregate` | Aggregates rows for ARRAY subqueries |
| `LimitOffset` | Applies LIMIT and OFFSET |
| `RecursiveFilter` | Handles self-referential policies |
| `Output` | Marks the query output |

## Supported SQL Features

Currently implemented:

- SELECT with column projections
- FROM with optional table aliases
- JOIN (INNER only)
- WHERE with equality conditions (AND)
- LIMIT and OFFSET
- ARRAY subqueries: `ARRAY(SELECT ...)`

### Not Yet Implemented

- ORDER BY (results ordered by ObjectId/insertion time)
- GROUP BY and HAVING
- Aggregate functions (COUNT, SUM, AVG, MIN, MAX)
- LEFT/RIGHT/OUTER JOINs
- Complex subqueries

## Performance

Incremental queries track dependencies at the row level. When a row changes:

1. The system identifies affected query graph nodes
2. Only those nodes recompute
3. Deltas propagate through the graph
4. Final results reflect the change
