## Overview

`col.bytes()` declares a column that stores arbitrary binary data. It maps to `Uint8Array`
in TypeScript and syncs as a binary blob alongside all other row data — no separate file
storage API or CDN required.

Binary columns are first-class: they participate in the same subscription and write APIs as
any other column, and are replicated to every connected client automatically.

## Defining a binary column

```ts
// schema/current.ts
import { table, col } from "jazz-tools";

table("recordings", {
  name: col.string(),
  audio: col.bytes(), // stores raw audio file contents
  duration_ms: col.int(),
});
```

The generated TypeScript interface reflects this:

```ts
// schema/app.ts (auto-generated)
export interface Recording {
  id: string;
  name: string;
  audio: Uint8Array; // typed as Uint8Array throughout
  duration_ms: number;
}
```

## Writing binary data

Read a `File` or `ArrayBuffer` and pass it directly as `Uint8Array`:

```ts
async function uploadRecording(name: string, file: File) {
  const buffer = await file.arrayBuffer();
  await db.insert(
    app.recordings,
    {
      name,
      audio: new Uint8Array(buffer),
      duration_ms: 0,
    },
    { tier: "edge" },
  );
}
```

## Reading binary data

Binary columns are returned as `Uint8Array` and can be used directly with any Web API that
accepts typed arrays:

```ts
async function playRecording(recording: Recording) {
  const audioCtx = new AudioContext();
  const audioBuffer = await audioCtx.decodeAudioData(recording.audio.buffer);
  const source = audioCtx.createBufferSource();
  source.buffer = audioBuffer;
  source.connect(audioCtx.destination);
  source.start();
}
```

## Caching on the client

Binary blobs are persisted in OPFS alongside the rest of the database, so they survive page
reloads without a network round-trip. On a return visit, `db.all` with the default `worker`
tier serves the blob from local storage immediately.

```ts
// First visit: fetches from edge and caches locally
// Subsequent visits: served instantly from OPFS, no network needed
const recordings = await db.all(app.recordings);
```

To guard against duplicate seeding on a first visit across concurrent clients, use
`{ tier: "global" }` — see [Reading Data: Seeding and one-time setup](/docs/reading-data#seeding-and-one-time-setup).

## Size considerations

Binary blobs are replicated in full to every subscriber. Keep individual blobs reasonable
for the expected connection quality of your users. For very large assets (video files,
high-resolution images), consider storing only metadata in Jazz and loading the asset from
a CDN by URL.
