  can find them.

## Inherited access policies (`allowedTo.*`)

## Share-based access

Sometimes access isn't determined by ownership or a parent relationship, but by a separate "shares" table. Use `policy.<table>.exists.where(...)` to check whether a matching row exists in another table.

## Policy enforcement and request context

Every policy is evaluated against the current user's session. On the frontend, this is the authenticated user. In a backend handler, create a scoped session per request so queries run with the right identity.
