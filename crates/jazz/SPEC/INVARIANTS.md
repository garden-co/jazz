# Invariant Registry

## Overview

This chapter is the out-of-band invariant registry for stable `INV-*` anchors. It maps each invariant to its status, coverage, enforcing tests, and implementation anchors.

Invariant digest: the complete one-line-per-invariant registry is the table in Details; duplicating hundreds of rows here would make onboarding worse and risk table drift.

## Details

Out-of-band spec-to-code convergence registry for stable `INV-<AREA>-<n>` anchors, per `SPEC/1_intro.md`. This file records where chapter invariants are tested or implemented; numbered chapters remain the normative source, while appendix IDs (`INV-DISC-*`, `INV-GLOS-*`, etc.) are guidance anchors rather than conformance requirements.

Each row carries the two orthogonal axes defined in `SPEC/1_intro.md` §1.3:

- **status** — design standing: `now` (in force in the current implementation; the contract today), `target` (a committed design destination, not yet in force), `open` (the design itself is unsettled — see the owning chapter's _Open questions_), or `prov` (true in the current implementation but not a hard requirement; a conformant implementation may differ).
- **coverage** — `✓` if an enforcing test is identified, `untested` otherwise.

Every `now` invariant trends toward coverage `✓`; an untested `now` is visible debt. `target` and `open` rows are coverage-exempt by definition. Values are transcribed from the research-stage assessment and approximate until a CI check makes them authoritative by verifying each referenced test exists.
